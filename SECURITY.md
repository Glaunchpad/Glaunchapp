# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅        |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security issue in GLaunch (the hosted product at [glaunch.world](https://glaunch.world/)), contact us responsibly:

1. **Email / X DM:** Reach out via [@Glaunchworld on X](https://x.com/Glaunchworld) with subject `Security Report`
2. Include steps to reproduce, impact assessment, and any proof-of-concept
3. Allow up to **72 hours** for an initial response

We will acknowledge valid reports and work on remediation. We ask that you do not disclose the issue publicly until we have had a reasonable window to address it.

## Scope

**In scope**

- glaunch.world and its subdomains
- Authentication and wallet flows
- On-chain actions initiated through GLA

**Out of scope**

- Third-party services (Privy, Flaunch, etc.) — report to those vendors directly
- Social engineering
- Denial-of-service against public endpoints

## Safe Harbor

Good-faith security research that follows this policy will not be pursued legally by Glaunchpad, provided you do not access, modify, or exfiltrate user data beyond what is necessary to demonstrate the issue.
